# Railway Deployment Guide - Kenge ISP Billing System

## 📋 Project Overview

This is a full-stack ISP billing system with 3 services:
- **Backend**: Next.js API with Prisma + PostgreSQL
- **Frontend**: React Vite SPA (Material-UI)
- **Landing Page**: Next.js marketing site (Tailwind CSS)

## 🚀 Quick Deploy to Railway

### Step 1: Create Railway Account
1. Go to [railway.app](https://railway.app)
2. Sign up/login with GitHub
3. Create a new project

### Step 2: Deploy Backend Service
1. Click "New Project" → "Deploy from GitHub"
2. Select your repository
3. Railway will auto-detect services, but set root directory to `backend/`
4. Add PostgreSQL plugin (Railway dashboard → Add → Database → PostgreSQL)
5. Set environment variables in Railway dashboard:

```
DATABASE_URL=postgresql://[auto-generated-by-railway]
JWT_SECRET=[generate-random-64-char-string]
NODE_ENV=production
GOOGLE_CLIENT_ID=[your-google-oauth-client-id]
CORS_ORIGIN=[your-frontend-url-after-deployment]
```

### Step 3: Deploy Frontend Service
1. Add new service from same repo
2. Set root directory to `frontend/`
3. Set environment variables:

```
VITE_API_URL=[your-railway-backend-url]
VITE_GOOGLE_CLIENT_ID=[same-as-backend]
```

### Step 4: Deploy Landing Page Service
1. Add new service from same repo
2. Set root directory to `landing-page/`
3. No environment variables needed (static site)

### Step 5: Run Database Migrations
After backend deploys, run migrations:
```bash
railway run npx prisma migrate deploy
railway run npx prisma db seed
```

## 🔧 Environment Variables Required

### Backend Service
```
DATABASE_URL=postgresql://user:pass@host:port/db  # Auto-generated by Railway PostgreSQL
JWT_SECRET=your-64-character-random-string       # Generate: openssl rand -hex 32
NODE_ENV=production
GOOGLE_CLIENT_ID=your-google-oauth-id
CORS_ORIGIN=https://your-frontend.railway.app    # Set after frontend deploys
```

### Frontend Service
```
VITE_API_URL=https://your-backend.railway.app
VITE_GOOGLE_CLIENT_ID=same-as-backend
```

### Landing Page
No environment variables required.

## 🐳 Docker Configuration

All services include optimized Dockerfiles:

- **Backend**: Multi-stage build, production dependencies only
- **Frontend**: Nginx serving static files with SPA routing
- **Landing Page**: Standard Next.js production build

## 📊 Database Setup

1. Railway PostgreSQL plugin auto-creates database
2. Connection string is automatically set as `DATABASE_URL`
3. Run migrations after deployment:
   ```bash
   railway run npx prisma migrate deploy
   ```
4. Seed initial data:
   ```bash
   railway run npx prisma db seed
   ```

---

## 🔗 Service URLs

After deployment, you'll get Railway URLs:
- Backend: `https://backend-name.railway.app`
- Frontend: `https://frontend-name.railway.app`
- Landing: `https://landing-name.railway.app`

Update `CORS_ORIGIN` in backend with frontend URL.

## 🚨 Troubleshooting

### Build Fails
- Check Railway build logs
- Ensure all dependencies are in package.json
- Verify Dockerfiles are correct

### Database Connection Issues
- Verify `DATABASE_URL` is set
- Check Railway PostgreSQL plugin is attached
- Run: `railway run npx prisma db push`

### Frontend Can't Reach Backend
- Check `VITE_API_URL` is correct
- Verify CORS settings in backend
- Check Railway service URLs

### Environment Variables Missing
- Set all required variables in Railway dashboard
- Restart services after adding variables

## 📝 Development vs Production

### Development (Local)
- Backend: `npm run dev` (port 3001)
- Frontend: `npm run dev` (port 5175)
- Landing: `npm run dev` (port 3000)

### Production (Railway)
- Backend: Docker container on port 3001
- Frontend: Nginx on port 80 serving static files
- Landing: Docker container on port 3000

## 🔒 Security Notes

- Generate unique `JWT_SECRET` for each deployment
- Set `CORS_ORIGIN` to your specific domain (not "*")
- Use Railway's built-in SSL certificates
- Keep Google OAuth credentials secure
